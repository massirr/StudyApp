Route Map

No react-router. App.tsx matches window.location.pathname manually.

URL Renders Notes
/ DashboardPage Topic list, stats, resume button
/topics/:slug TopicPage Subtopics, notes, "Open Quiz" link. 404 if slug unknown.
/quiz?topic=:slug QuizPage (wrapper)
→ components/quiz/QuizPage		 Reads ?topic from URL. If omitted, loads all questions.
/topic/:slug — Redirects to /topics/:slug
* NotFoundPage Catch-all 404
Navigation

All navigation uses plain <a href="…"> tags or window.history.replaceState(). The browser's popstate event triggers re-renders on back/forward.

Data Flow — Progress (Persistent)
🗄 LocalStorage
Key: studyapp_progress
JSON blob: completedTopicIds, lastVisitedTopicSlug, preferences, version
on app load — progressStorage.loadProgressState()
⚛ ProgressContext
React useState holds ProgressState. Provider wraps entire app in main.tsx. Exposes: progress, toggleTopicComplete, setLastVisitedTopic, resetProgress
consumed via useProgress() hook
DashboardPage
reads progress, toggles complete, resets
TopicPage
calls setLastVisitedTopic on mount
on every state change — progressStorage.saveProgressState()
🗄 LocalStorage
Written back immediately. Persists across browser sessions.
Data Flow — Static Study Content

Topics, questions, notes, and sources are hardcoded TypeScript arrays in src/data/.
They are imported directly — no fetch, no database.
All source URLs are validated at build time by contentValidation.ts to be official Microsoft docs.

Quiz Logic Flow
